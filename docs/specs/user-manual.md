# uniTerm 用户使用手册

## 1. 产品简介

uniTerm 是一款跨平台的多标签页终端工具，支持 SSH 远程登录和 SFTP 文件传输。它集成了 AI 助手功能，能够在您当前激活的终端会话中执行命令、读取输出并协助完成各类任务。

### 主要特性
- **多标签页**：支持多会话切换
- **SSH 终端**：基于 xterm.js 的全功能终端模拟器，支持颜色、Unicode、VT100 控制序列
- **SFTP 文件管理**：内置文件传输客户端，支持上传下载
- **AI 助手**：类 Claude Code 的对话式 AI，可自动或经确认后在终端中执行 shell 命令
- **跨平台**：Windows、macOS、Linux 均可运行（Windows 版本约 14MB）
- **体积小巧**：单个可执行文件，无任何外部依赖，点击即用

## 2. 界面概览

启动 uniTerm 后，主界面分为三个区域：

```
+--------------------------------------------------------+-----------+
| [文件] [新建连接] [AI]                          [设置]  |           |
+--------------------------------------------------------+           |
| 连接列表  |                                              |  AI 边栏   |
| - Host A  |           终端主区域 / SFTP 文件管理器          |  (可折叠)  |
| - Host B  |                                              |           |
| ...       |                                              |           |
+--------------------------------------------------------+-----------+
```

| 区域 | 位置 | 说明 |
|------|------|------|
| 顶部工具栏 | 最上方 | 新建连接、切换 AI 边栏、设置按钮 |
| 连接列表 | 左侧 | 保存的 SSH/SFTP 连接配置，点击快速连接 |
| 主区域 | 中间 | 终端标签页或 SFTP 文件管理器 |
| AI 边栏 | 右侧 | 可折叠的 AI 对话界面 |

## 3. 连接管理

### 3.1 新建连接

点击顶部工具栏的 **"新建连接"** 按钮，打开连接配置表单：

| 字段 | 说明 |
|------|------|
| 名称 | 为该连接取一个显示名称（如 "生产服务器"） |
| 类型 | SSH（终端）或 SFTP（文件传输） |
| 主机 | 服务器 IP 地址或域名 |
| 端口 | 默认 SSH 为 22 |
| 用户名 | 登录账号 |
| 认证方式 | 密码、私钥、或 SSH Agent |

填写完成后，点击 **"连接"** 即可建立会话；点击 **"仅保存"** 则将配置添加到左侧连接列表而不立即连接。

### 3.2 使用连接列表

左侧连接列表保存了您常用的服务器配置：
- **点击连接**：直接建立会话并打开新标签页
- **删除连接**：将鼠标悬停在连接项上，点击删除图标移除配置

连接配置存储在应用数据目录中（Windows：`%AppData%/uniTerm/`），随应用启动自动加载。

### 3.3 终端标签页

每个 SSH 连接对应一个终端标签页：
- 右键点击标签可关闭当前标签或关闭其他标签
- 连接意外断开时，标签状态变为错误并提供**"重试"**按钮

## 4. 终端使用

### 4.1 基本操作

uniTerm 的终端基于 xterm.js，支持完整的 VT100 控制序列：
- 支持颜色输出、光标移动、清屏等标准终端操作
- 支持 Unicode 字符显示
- 输入延迟低于 50ms（本地网络环境下）

## 5. AI 助手

### 5.1 打开 AI 边栏

点击顶部工具栏的 **"AI"** 按钮，或点击边栏标题栏的关闭按钮切换显隐。AI 边栏默认宽度为 360px，可通过拖拽左侧边框调整宽度（范围 240px ~ 800px）。

### 5.2 配置 AI

首次使用 AI 功能前，需要配置 LLM 服务：

1. 点击边栏标题栏的 **设置（齿轮）图标**
2. 在弹出的设置对话框中填写：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| API Key | LLM 服务的 API 密钥 | sk-xxxxxxxx |
| Base URL | API 基础地址 | `https://api.openai.com/v1` |
| Model | 模型名称 | gpt-4o、claude-3-sonnet 等 |
| Protocol | API 协议类型 | OpenAI 或 Anthropic |

3. 点击 **"Save"** 保存配置（配置将持久化到本地文件，重启后仍然有效）

> **支持的 LLM 服务商**：通过配置不同的 Base URL 和 Model，可以对接 OpenAI、Anthropic、OpenRouter、Ollama（本地部署）等任意 OpenAI 兼容接口。

### 5.3 对话操作

#### 发送消息
- **Enter**：发送消息
- **Shift + Enter**：在输入框中插入换行符
- **Ctrl + Enter**：发送消息（备选）

#### 消息类型
AI 边栏中显示三种角色的消息：

| 角色 | 头像 | 说明 |
|------|------|------|
| You | 灰色头像 | 您发送的消息 |
| AI | 蓝色头像 | AI 助手的回复，支持 Markdown 格式渲染 |
| Tool | 绿色头像 | 仅在 Debug 模式下可见，显示工具调用详情 |

AI 消息中的 Markdown 支持：代码块、粗体、斜体、链接、列表、表格、分隔线等。

#### Session 管理

边栏标题栏下方有 Session 下拉选择器，用于管理对话历史：
- 当前正在进行的对话显示为 "New Chat" 或自动命名的标题
- 点击下拉菜单可查看历史会话列表，选择即可加载继续对话
- 每个历史会话右侧显示最后更新时间（如 "2分钟前"）
- 点击 **"+ New Chat"** 创建新会话
- 将鼠标悬停在历史会话上，点击删除图标可移除该会话

**自动命名规则**：系统会自动取当前会话中第一条用户消息的前 20 个字符作为会话标题。

### 5.4 执行模式

AI 助手提供两种命令执行模式，通过边栏顶部的分段控制器切换：

| 模式 | 说明 |
|------|------|
| **Auto** | AI 自动执行命令，无需确认。适合您信任的任务，AI 可自主完成多轮执行。 |
| **Confirm** | AI 每执行一条命令前需您确认。适合需要谨慎操作的生产环境。 |

#### Auto 模式流程
```
您发送任务 → AI 分析并回复 → 如需执行命令，自动在终端运行
    → 读取命令输出 → 继续推理 → 循环直到任务完成
```

#### Confirm 模式流程
```
您发送任务 → AI 分析并回复 → 如需执行命令，显示 "Run"/"Skip" 按钮
    → 您点击 Run：执行命令 → 读取输出 → AI 继续推理
    → 您点击 Skip：跳过该命令 → 记录 "User rejected this command."
```

### 5.5 工具调用展示

当 AI 需要执行终端命令时，会在消息中以折叠面板形式展示 IN/OUT 块：

#### IN 块（命令输入）
- **样式**：绿色主题
- **头部**："IN" 标签 + 函数名（如 `execute_command`）+ 展开/折叠箭头
- **内容**：仅显示要执行的命令文本（提取自 JSON 参数中的 `command` 字段）
- **默认状态**：折叠

#### OUT 块（执行结果）
- **样式**：紫色主题
- **头部**："OUT" 标签 + 展开/折叠箭头
- **内容**：命令执行的 stdout/stderr 输出文本
- **默认状态**：折叠
- **最大高度**：300px，超出可滚动

#### 待审批状态（Confirm 模式）
当 AI 发起工具调用且当前为 Confirm 模式时：
- 在消息下方显示待审批区域：工具名 + 命令文本 + "Run" / "Skip" 按钮
- IN 块照常展示
- OUT 块不显示（尚未执行）

### 5.6 Debug 模式

勾选边栏顶部的 **"Debug"** 复选框可开启调试模式：
- 在对话中插入 debug 消息（id 前缀 `dbg-`），显示请求体、响应内容等调试信息
- Debug 消息在消息列表中可见，但**不会传入 LLM 对话上下文**
- 用于排查请求格式、响应解析、执行流程等问题

### 5.7 停止运行

当 AI 正在运行时（显示 "Thinking..."），输入框旁会出现 **"Stop"** 按钮：
- 点击 Stop 可中断当前 LLM 流式接收
- 若 AI 正准备执行命令，点击 Stop 将取消命令执行

## 6. 配置存储位置

uniTerm 的所有配置存储在以下位置：

| 平台 | 配置目录 |
|------|---------|
| Windows | `%AppData%/uniTerm/` |
| macOS | `~/Library/Application Support/uniTerm/` |
| Linux | `~/.config/uniTerm/` |

存储的文件包括：
- `connections.json`：保存的连接配置列表
- `ai-config.json`：AI 助手的 API Key、Base URL、Model、Protocol 配置
- `ai-sessions`（localStorage）：AI 对话历史（存储在浏览器本地存储中）

## 7. 常见问题

### Q: 为什么 AI 配置重启后不见了？
A: 早期的 uniTerm 版本使用浏览器 localStorage 存储 AI 配置，在 WebView2 环境中不可靠。请升级到最新版本，配置现在通过 Go 后端持久化到本地文件 `%AppData%/uniTerm/ai-config.json`，重启后仍然有效。

### Q: AI 不执行命令，只回复文本？
A: 请检查：
1. 当前是否有激活的终端标签页（AI 需要绑定到一个终端会话才能执行命令）
2. AI 设置中的 API Key、Base URL、Model 是否正确
3. 若提示 "请先在主窗口中打开一个终端会话"，请先建立 SSH 连接

### Q: 前端修改没有生效？
A: uniTerm 的 `wails dev` 直接从 `frontend/dist/` 目录加载前端资源。修改代码后需要执行 `cd frontend && npx vite build` 重新构建，然后重启应用才能看到变化。

### Q: 调整 AI 边栏宽度时终端卡顿？
A: 最新版本已优化此问题。若仍遇到卡顿，请确保更新到包含拖拽性能优化的版本。

### Q: Confirm 模式下点击 Run/Skip 没有反应？
A: 请确保：
1. 当前有激活的终端会话
2. 检查控制台是否有错误输出
3. 尝试重新编译并启动应用

### Q: 支持哪些 LLM 服务商？
A: 理论上支持所有提供 OpenAI 兼容接口的服务商，包括：
- OpenAI（GPT 系列）
- Anthropic（Claude 系列，通过后端协议转换）
- OpenRouter（聚合多种模型）
- Ollama（本地运行开源模型）
- 以及任何其他兼容 OpenAI Chat Completions API 的服务

### Q: 密码和私钥是否安全？
A: uniTerm 当前版本**不存储密码和私钥内容**。连接配置中保存的是私钥文件路径（而非密钥内容），每次连接时需要手动输入密码或密钥短语。后续版本计划接入操作系统 keychain 实现安全存储。
